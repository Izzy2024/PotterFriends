   TRY #1: Limpia caché del navegador
   ───────────────────────────────────
   [ ] Presiona Ctrl+Shift+Delete (Cmd+Shift+Delete en Mac)
   [ ] Marca "Imágenes y archivos en caché"
   [ ] Haz clic en "Limpiar datos"
   [ ] Recarga la página del foro
   [ ] Prueba de nuevo

   ─────────────────────────────────────
   [ ] Ve a Supabase → SQL Editor

       ✗ No → Vuelve a ejecutar el script SQL completo

   TRY #3: Cierra sesión y vuelve a entrar
   ────────────────────────────────────────
   [ ] Haz clic en "Cerrar sesión"
   [ ] Inicia sesión nuevamente
   [ ] Ve al foro
   [ ] Prueba crear una discusión

   TRY #4: Verifica tu sesión actual
   ─────────────────────────────────
   [ ] Abre consola del navegador (F12)
   [ ] Ve a pestaña "Console"
   [ ] Copia y pega esto:
       
       const { data } = await window.supabaseClient.auth.getUser();
       console.log('User:', data.user);
       console.log('Metadata:', data.user.user_metadata);
   
   [ ] Verifica que aparezca tu información

   [ ] Lee el mensaje de error
   [ ] ¿Dice "permission denied"?
       → Verifica que tienes permisos de admin en Supabase
   
   [ ] ¿Dice "table already exists"?
       → Esto está bien, continúa ejecutando el resto
   
   [ ] ¿Otro error?
       → Copia el mensaje de error y búscalo en Google
       → O contacta soporte de Supabase
